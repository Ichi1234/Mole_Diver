# Mole Diver

## Overview
Mole Diver is a mobile game developed as part of a university course project. The primary goal was to design and build a playable Android game using Android Studio, applying concepts covered in the course such as touch input handling, game loop implementation, UI design, and local data storage.

## Game Concept
The concept was built around a simple but engaging premise: a meteorite has struck the Earth and released a deadly poison gas that is slowly spreading downward through the soil. The only hope for saving the planet is to reach **Mole Jesus**, a powerful figure resting at the Earth's core.

The player controls a mole that must dig down through underground layers, outrunning the descending gas, collecting valuable materials along the way, and fighting enemies before the poison catches up.

This storyline gives the game a clear purpose and urgency beyond simply surviving. The poison gas mechanic creates constant pressure without requiring a timer, while the objective of reaching Mole Jesus provides a defined win condition that players can aim for across multiple runs.

## Controls
The controls were designed to be simple and accessible for mobile devices.

Instead of using a joystick or swipe gesture, the player steers the mole by tapping the **left or right side of the screen**, which gradually rotates the digging angle.

This allows the mole to:
- Dig straight downward
- Dig diagonally
- Curve back upward if needed

This control system adds a layer of strategy while keeping the gameplay intuitive and easy to learn.

## Progression System
Mole Diver uses a **run-and-upgrade progression system**.

Materials collected during a run are converted into currency. Players can then spend this currency in the **Shop** between runs to permanently upgrade the mole’s abilities.

Possible upgrades include improvements such as:
- Digging efficiency
- Movement capability
- Resource collection

This design ensures that even unsuccessful runs contribute to long-term progress, encouraging players to continue improving their character over multiple sessions.

## Project Goals
This project focuses primarily on gameplay mechanics and core systems, including:

- Touch input handling
- Game loop implementation
- Android UI design
- Local data storage for player progress

Additional cosmetic or social features were considered lower priority and may be added in future iterations.
